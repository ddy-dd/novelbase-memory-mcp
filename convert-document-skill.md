# convert-document — 小说文档分析技能

读取 Word 或 PDF 格式的小说，**像编辑一样分析这部作品的结构**——谁是谁、发生了什么、为什么、埋了什么、哪里是转折——然后把这些理解写成 `.md` 文件。

> 核心工作不是格式转换，是**文学分析**。你读到的每一段都要问自己：这段在推进什么？人物为什么这么做？这里是不是伏笔？

---

## 第 1 步：读文档

拿到路径后，先把文字提取出来。

**PDF**：Read 工具直接读。长文档分页读，避免截断。
**Word（.docx）**：按以下顺序试：

```bash
# 方案 A
pandoc "$FILE" -t plain 2>/dev/null
# 方案 B（pandoc 不可用时）
python3 -c "
import docx
for p in docx.Document('$FILE').paragraphs: print(p.text)
"
```

读完全文，**通读一遍再动笔**。读的时候关注：

## 第 2 步：梳理人物（最核心）

**目标**：搞清楚这小说里谁是谁，不要遗漏。

### 2.1 列出所有有名有姓的角色

通读后，列出每一个人物。包括：
- 大名、小名、字号、绰号（如林黛玉/颦儿/潇湘妃子）
- 身份（皇帝/丫鬟/将军/商人/道士）
- 跟谁是一家的（家族/师门/阵营）

> 注意：有些角色别称很多，都记录下来。典韦和许褚可能被并称"虎卫双雄"，但他们是两个人。

### 2.2 判断主角和配角

主角通常符合：
- 出场次数多
- 有独立的动机和目标
- 推动了关键情节
- 有完整的成长弧线

配角是：
- 为了衬托主角存在
- 推动某个具体情节后消失
- 没有独立的动机线

### 2.3 判断人物弧光

人物的性格或处境在故事过程中有变化吗？

```
林黛玉：    寄人篱下 → 绽放才情 → 为情所困 → 泪尽而逝
            （有弧光，有明显的变化轨迹）

贾母：      全府至尊 → 晚年衰败
            （有弧光，地位变化）

平儿：      通房丫鬟 → 最终扶正
            （有弧光，身份变化）
```

没有弧光的角色（功能性角色）标记即可，不需要深挖。

### 2.4 识别人物关系网

不光要记录"贾宝玉认识林黛玉"，要记录**关系的性质**：

```
贾宝玉 ── 林黛玉：表兄妹 + 恋人 + 精神知己
贾宝玉 ── 薛宝钗：表姐弟 + 夫妻（金玉良缘）但无情
贾宝玉 ── 贾政：父子 + 对立（仕途 vs 真情）
林黛玉 ── 紫鹃：主仆 + 姐妹情
```

注意关系是**动态演变**的：初遇时什么样，后来变成什么样。

## 第 3 步：梳理情节结构

### 3.1 划分四阶段

绝大多数小说都可以分：

```
开端：   介绍世界观、主要人物出场、核心矛盾埋下
发展：   矛盾展开、各方角力、冲突升级
高潮：   核心矛盾爆发、决定性事件
结局：   尘埃落定、人物归宿
```

对章回体小说（如《三国演义》），看回目自然就能分。

### 3.2 每章在推进什么

给每章/每回一句话概括它在整本书中的功能：

```
第1回：  介绍贾府全盛背景 + 贾宝玉出生（定调）
第3回：  林黛玉进贾府（核心人物集合）
第27回： 黛玉葬花（人物性格的关键展示）
```

### 3.3 揪出核心矛盾

一本书通常有 1-2 个核心矛盾，若干次要矛盾：

```markdown
## 冲突
- 贾宝玉 vs 贾政：仕途经济 vs 真情自由（核心矛盾）
- 林黛玉 vs 薛宝钗：爱情竞争者（次要矛盾）
- 贾府 vs 朝廷：盛极而衰的必然（宏观矛盾）
```

## 第 4 步：追踪伏笔和反转

### 4.1 找伏笔

伏笔的特征：
- 前期出现时**看似无关紧要**
- 后期揭开时**恍然大悟**
- 通常是：物品、对话、梦境、判词、预言

```
红楼梦的伏笔：
  第5回 贾宝玉梦游太虚幻境 → 看到金陵十二钗判词
  判词"玉带林中挂，金簪雪里埋" → 预示黛玉宝钗的命运
  这伏笔跨越了 100 多回
```

### 4.2 找反转

反转的特征：
- 读者/角色的预期和实际结果相反
- 通常伴随"原来如此"的效果

```
反转示例：
  预期：贾府有元春撑腰，荣华富贵可保
  实际：元春失宠病死，贾府被抄

  预期：刘姥姥就是个搞笑村妇
  实际：贾府败落后她救了巧姐
```

### 4.3 判断呼应关系

有些情节前后呼应：

```
前：宝玉第一次摔玉（第3回）→ 后：宝玉最终弃玉出家（第120回）
前：刘姥姥进大观园要钱（第6回）→ 后：刘姥姥救巧姐报恩（第119回）
```

这些都是 FORESHADOWS 边可以连接的。

## 第 5 步：写成 .md 文件

### 5.1 目录结构

```
novelbase/
├── 世界观.md
├── 第1回-章节名.md
├── 第2回-章节名.md
├── ...
```

### 5.2 世界观.md（全局）

```markdown
# 世界观设定

## 环境
- 时代：清朝乾隆年间
- 社会：封建大家族，等级森严
- 风俗：元宵节、中秋节、诗社
- 文化：诗词、戏曲、园林、茶道

## 情节脉络
- 开端：1-5回，贾府全盛，人物出场
- 发展：6-80回，矛盾积累，悲剧萌芽
- 高潮：81-100回，抄检大观园，家族衰败
- 结局：101-120回，树倒猢狲散，宝玉出家

## 冲突
- 贾宝玉 vs 贾政 → 仕途理想与个人自由的冲突
- 林黛玉 vs 薛宝钗 → 爱情与婚姻的竞争
- 贾府 vs 朝廷 → 盛极而衰的必然

## 角色
- 贾宝玉：怡红公子、宝二爷（公子，叛逆多情，外貌：面若中秋之月，动机：追求真情，心理：厌恶仕途，结局：出家为僧）
- 林黛玉：颦儿、潇湘妃子（小姐，多愁善感，外貌：闲静时如姣花照水，动机：追求知己之爱，心理：敏感多疑，结局：泪尽而逝）
- ...所有主要角色

## 关系
- 贾宝玉 → 林黛玉：表兄妹，木石前盟
- 贾宝玉 → 薛宝钗：表姐弟，金玉良缘
- ...所有核心关系

## 地点
- 大观园：贾府为元春省亲所建
- ...所有重要地点

## 伏笔
- 判词 → 人物结局：金陵十二钗判词预示各人命运
- 刘姥姥进大观园 → 贾府败落后救巧姐

## 反转
- 预期：金玉良缘 → 实际：宝玉出家
- 预期：贾府长盛 → 实际：家破人亡

## 标签
- 情节：元妃省亲、抄检大观园、黛玉葬花
- 文化：中医药、诗词、建筑、饮食
- 时空：乾隆年间、大观园、潇湘馆
```

### 5.3 每章的文件

```markdown
# 第3回 托内兄如海荐西宾

## 角色
- 林黛玉：颦儿、潇湘妃子（小姐，多愁善感）
- 贾母（老夫人，慈祥和蔼）
- 王熙凤：凤辣子（管家奶奶，泼辣能干）

## 地点
- 荣国府：贾府正宅
- 贾母院：贾母住处

## 关系
- 林黛玉 → 贾母：祖孙
- 林黛玉 → 王熙凤：表嫂嫂

## 标签
- 情节：关键人物出场
- 文化：大家族的规矩礼仪

## 原文摘录
> 黛玉便知这方是正经正内室，一条大甬路，直接出大门的。
```

### 5.4 格式规则

| 字段 | 格式 | 例子 |
|------|------|------|
| 角色行 | `- 本名：别名（身份，性格，外貌：，动机：，心理：，结局：）` | `- 林黛玉：颦儿（小姐，多愁善感，外貌：闲静时如姣花照水，结局：泪尽而逝）` |
| 地点行 | `- 名称：描述` | `- 潇湘馆：林黛玉住所，多竹` |
| 关系行 | `- A → B：描述` | `- 贾宝玉 → 林黛玉：表兄妹，木石前盟` |
| 伏笔行 | `- 来源 → 目标：描述` | `- 判词 → 人物结局：金陵十二钗判词预示命运` |
| 反转行 | `- 预期：xxx → 实际：yyy` | `- 预期：金玉良缘 → 实际：宝玉出家` |
| 标签行 | `- 分类：标签1、标签2` | `- 情节：元妃省亲、抄检大观园` |
| 环境行 | `- 类别：描述` | `- 时代：清朝乾隆年间` |

**注意**：
- 文档中没有的信息**不要编造**。例如原文没写角色结局，就省略 `结局：` 字段
- 角色括号内用 `，` 分隔不同字段，特殊字段用 `外貌：`、`动机：`、`心理：`、`结局：` 前缀标记
- 每个文件都要写 `## 角色`，即使该章没有新角色出场，也要列出本章出现的角色
- 每章写 `## 原文摘录`，选 1-2 段最有代表性的原文（用于日后溯源）

## 第 6 步：输出

```bash
mkdir -p novelbase/
```

把文件写进去，完成后给用户总结：

```
📋 分析结果：
  总章节：120 回
  角色数：约 80 人
    其中主角：6 人（贾宝玉、林黛玉、薛宝钗、王熙凤、贾母、贾政）
    有完整弧光：12 人
  核心矛盾：1（真情 vs 仕途）
  伏笔：8 处（最主要：太虚幻境判词）
  反转：3 处（最主要：金玉良缘→宝玉出家）
  标签覆盖：情节12类、文化8类、时空5类

📁 输出文件：novelbase/（120 个章节文件 + 世界观.md）

➡️ 下一步：cargo run -- cli import novelbase/ --project 红楼梦 --source original
```
